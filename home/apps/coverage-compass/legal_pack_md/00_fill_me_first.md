# FILL ME FIRST — Legal Pack Setup (v1.1 — Blue Button removed)

This folder contains a comprehensive, **non‑marketing / non‑lead‑gen** legal and disclosure pack for a consumer decision‑support app that helps people compare **Original Medicare + Medigap** tradeoffs.

## What changed in v1.1 (this build)
This version **removes all Blue Button / Blue Button 2.0 / CMS claims‑API assumptions**.
- The documents now assume **no connection to Medicare.gov accounts** and **no claims data pull**.
- If you later add claims import (or any OAuth connection to Medicare.gov), you will need a new addendum set: authorization language, data minimization rules, security controls, and an updated privacy notice.

## How to use these documents
1. Replace bracketed placeholders like **[APP NAME]** and **[COMPANY NAME]**.
2. Delete any sections you don’t want (e.g., arbitration, class action waiver) and have counsel review.
3. Publish these documents in‑app (Settings → Legal) and on your website.

## Placeholder keys to search/replace

- [COMPANY NAME]
