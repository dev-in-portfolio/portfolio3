# Alibi — Logged & Verified — Complete Build

## What this is
Alibi is a trust-aware kitchen inventory and verification system. Items move through explicit states (Draft → Locked → Verified) so users can see not just counts, but confidence and auditability.

## Why this exists
Kitchens run on interruptions, handoffs, and human error. Alibi exists to treat uncertainty and verification as first-class—so the system reflects reality instead of pretending data is perfect.

## What this proves
- Stateful system design beyond CRUD
- Human-error-aware workflow modeling
- Trust UX (confidence, verification, reconciliation)
- Operational product design under pressure
- Clear state transitions and recovery paths
## One‑liner
**Kitchen inventory with an audit trail** — every change is logged, verified, and reviewable.

## What it demonstrates
- Inventory CRUD with fast search/filter
- Event logging / verification UX
- “Reality-first” product thinking (inventory isn’t real unless it’s counted)
