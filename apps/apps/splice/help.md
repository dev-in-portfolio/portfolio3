# Splice - User Guide

Splice is a sequencing tool for arranging short clips into a paced story, with beat markers, timing controls, and local project export.

## Features
- Visual clip sequencer timeline with a runtime summary.
- Beat markers for intro, build, reveal, proof, and CTA moments.
- Timing controls for trim ranges, transition intent, and hold beats.
- Per-beat notes that carry through into project export.
- Pacing diagnostics that flag missing structure or risky beat lengths.
- Exportable sequence brief for editors, collaborators, or client review.
- Modern, dark-themed UI with grid alignments.
- Neon glow effects for active elements.

## Step-by-Step Usage
1. Open the Splice app.
2. Add or import media clips into the workspace.
3. Add clips to the timeline and set the trim range for each beat.
4. Mark each clip as intro, build, reveal, proof, or CTA, then set transition intent and any hold time after the beat.
5. Use beat notes to document what should happen in that moment or what the next cut should emphasize.
6. Check the pacing readout for structure or rhythm issues, then export the sequence brief if you need to share the beat plan outside the app.
7. Export or save your sequence when finished.

## Troubleshooting
- If drag-and-drop feels unresponsive, try refreshing the page.
- Visual elements like neon glows may appear different depending on your monitor's contrast settings.
- If the timeline scrolls out of view, use the custom scrollbars or trackpad to navigate back.
- Transition mode is planning metadata right now; hold timing affects preview pacing directly, while transition type mainly improves project structure and export notes.
- Hold timing now affects both sequence playback and the local render path, so pacing should stay more consistent between preview and export.
